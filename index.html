Financial Analytics Dashboard

A company deep-dive dashboard for finance leaders. Built on a star schema with DimCompany, DimDate, and FactFinancials, plus a dedicated Measures table. Designed for single-company analysis with sector peer benchmarks.

Key Features

  • Peer Benchmarking: Compare a selected company to sector averages using dynamic DAX.
  • Profitability Waterfall: Visualize movement from Revenue → Gross Profit → Operating Profit → Net Income.
  • Scatter Plot (Peers): Evaluate companies on metrics like Margin vs Growth; tooltip shows key KPIs.
  • Measure Catalog: Organized folders: Profitability, Liquidity & Leverage, Efficiency, Valuation.
  • Slicers: Company (single-select) and Date; all visuals respect context.

Tech Stack

Power BI · DAX · SQL · Python (prep) · Star Schema

Marketing Analytics Dashboard

End-to-end marketing analytics on the Olist E-commerce dataset. Built on a star schema (FactSales, DimCustomer, DimDate, DimProduct, DimGeography) with a dedicated measures table. Focus areas: cohort retention, RFM, CLV, and campaign ROAS/CAC.

Key Features

  • Cohort & Retention: Retention heatmap over 12+ months with weighted/unweighted averages and cohort size.
  • CLV Modeling: Expected future profit, churn probability, CLV distribution and top customers/countries by CLV.
  • RFM Segmentation: Percentile scoring (1–5) to identify Champions, Loyal, Potential Loyalists, At Risk.
  • Campaign Performance: ROAS, Profit ROAS, CAC; efficiency bubbles (spend vs revenue) and new vs returning revenue.
  • Country & Product Insights: Pareto charts, contribution by month, and scatter (revenue vs units; bubble = profit).
  • Seasonality: YoY/MoM trends and weekday × month patterns.

Tech Stack

Power BI · DAX · Python (prep) · SQL · Olist Dataset

Loan Default Prediction Model

Project Overview

Developed a machine learning classification model to predict loan default probability, helping financial institutions minimize losses and optimize their review processes for loan applications.

Technical Approach

  • Data Processing: Handled missing values, outliers, and performed feature scaling
  • Feature Engineering: Created new features based on borrower financial patterns and loan characteristics
  • Model Selection: Tested multiple algorithms (Random Forest, XGBoost, Logistic Regression)
  • Optimization: Hyperparameter tuning using GridSearchCV
  • Validation: Cross-validation with stratified k-fold

Business Impact

  • Reduced default rates through early risk identification
  • Optimized manual review processes
  • Minimized opportunity costs from false rejections
  • Improved loan portfolio quality
